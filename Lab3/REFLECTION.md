I ensured unique keys for list items by using the crypto.randomUUID() method when creating new task objects in the state, instead of array indexes.  I kept running into issues with adding, changing, filtering, and removing items, so I asked Google Gemini, and it was recommended using this method to assign a permanent and unique ID to each task.
The main consideration I had to make was using derived state instead of creating a separate piece of state for the filtered tasks. I handled state updates for status changes by using the .map() method so that React would know when data changed.
One challenge I encountered when implementing conditional rendering was correct the empty state, because when all tasks were deleted the app was broken and presented a blank screen below the 'No task found' message. I consulted Google Gemini to add the 'Add Task' button function.
